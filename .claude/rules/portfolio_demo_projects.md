## Demo Project 1 — Real-Time AI Sports Commentator

### Concept
User uploads a sports video clip and selects a commentary style (e.g., "ESPN anchor", "British football pundit", "hype Reddit thread"). The app analyzes the video and generates spoken commentary in that style.

### Tech Stack
| Layer | Tool |
|-------|------|
| Frontend | Next.js + Tailwind — video upload UI, style selector dropdown |
| Backend | FastAPI — video → frame extraction + audio separation |
| Vision | GPT-4o or Gemini Vision — scene understanding from sampled frames |
| LLM | Claude API — generates commentary script in chosen style |
| TTS | ElevenLabs API — converts commentary script to speech |
| Hosting | Vercel (frontend) + Render or Modal (backend) |

### Key Notes
- Sample every N frames rather than processing the full video — keeps latency manageable
- ElevenLabs voice cloning can add style differentiation per commentator persona
- Output: audio file the user can play inline + transcript shown on screen

## Demo Project 2 — Vibe Check Interview Coach ⭐ (Top Priority)

### Concept
User answers an interview question on camera. The app simultaneously analyzes facial emotion, speech patterns (filler words, pacing), and answer quality — then gives a scored coaching report.

### Why This Is The Priority Build
- Direct evolution of Mohammed's existing **AffectSync** project (DeepFace + Whisper already built)
- Multimodal (face + voice + language) — hits Mohammed's exact stated technical focus
- Immediately relatable to every CS student or recruiter who visits the portfolio
- Demos live in the browser with zero explanation needed

### Tech Stack
| Layer | Tool |
|-------|------|
| Frontend | Next.js + Tailwind — webcam + mic capture via WebRTC MediaRecorder API |
| Face Analysis | DeepFace or MediaPipe — emotion + eye contact detection (reuse from AffectSync) |
| Speech | OpenAI Whisper API — transcription, filler word detection ("um", "uh", "like") |
| LLM | Claude API — evaluates answer quality, generates coaching feedback |
| Scoring UI | Recharts or D3 — radar/spider chart: Confidence, Clarity, Tone, Eye Contact, Pacing |
| Backend | FastAPI — orchestrates DeepFace + Whisper + Claude calls |
| Hosting | Vercel (frontend) + Render or Modal (backend — DeepFace needs Python server) |

### Key Notes
- Record a short clip (30–90 sec), then process after recording ends — avoids real-time streaming complexity
- Output: numerical scores per dimension + paragraph coaching feedback per dimension
- Optional stretch: "Improved Answer" suggestion generated by Claude

## Demo Project 3 — AI Research Paper → Podcast

### Concept
User pastes an ArXiv paper URL. The app fetches the paper, generates a conversational two-host podcast script debating and explaining it, and produces an audio file with two distinct voices.

### Tech Stack
| Layer | Tool |
|-------|------|
| Frontend | Next.js — ArXiv URL input field, audio player UI, transcript viewer |
| Paper Fetching | ArXiv API + PyMuPDF — fetch and parse PDF text |
| LLM | Claude API — generates two-host conversational podcast script |
| TTS | ElevenLabs API — two distinct voice profiles, one per host |
| Audio Stitching | pydub (Python) — interleaves the two voice tracks into one audio file |
| Backend | FastAPI — orchestrates the full pipeline |
| Hosting | Vercel (frontend) + Render (backend) |

### Key Notes
- Limit paper ingestion to abstract + introduction + conclusion for speed (Claude context window aware)
- Two host personas: one "explainer" voice, one "skeptic/questioner" voice — makes it feel like a real podcast
- Given Mohammed's research background (medical imaging, deep learning), this demo feels authentic

## Demo Project 4 — Roast My Resume

### Concept
User uploads their resume PDF and pastes a job description. Claude tears it apart — ATS score, keyword gaps, phrasing issues — then produces a rewritten version side-by-side.

### Tech Stack
| Layer | Tool |
|-------|------|
| Frontend | Next.js + Tailwind — PDF upload + job description textarea, side-by-side output UI |
| PDF Parsing | PyMuPDF or pdfplumber (Python) — extracts resume text from uploaded PDF |
| LLM | Claude API — ATS scoring, keyword gap analysis, harsh roast, full rewrite |
| Diff Highlighting | react-diff-viewer or custom CSS — highlights changes between original and rewrite |
| Backend | FastAPI or Next.js API route — handles PDF parsing + Claude calls |
| Hosting | Vercel (can be full-stack with Next.js API routes — no separate backend needed) |

### Key Notes
- Mohammed has personally built this workflow for himself — the prompt engineering is already in his head
- No third-party ATS tool needed — Claude handles scoring via prompt engineering
- This is the most self-contained project (no webcam, no audio) — fastest to ship

## Demo Project 5 — Live Multilingual Emotion Subtitle Generator

### Concept
User points their webcam at someone speaking in any language. The app transcribes speech, translates it in real time, and overlays subtitles with the speaker's detected emotional state (e.g., "😤 Frustrated — 'I don't understand why this keeps happening'").

### Tech Stack
| Layer | Tool |
|-------|------|
| Frontend | Next.js — webcam stream via WebRTC, real-time subtitle overlay on video |
| Speech | OpenAI Whisper API (streaming or chunked) — live transcription |
| Translation | DeepL API or Google Translate API — multilingual output |
| Emotion Detection | DeepFace or MediaPipe — runs on periodic frame snapshots |
| Subtitle Rendering | Canvas API or CSS overlay positioned on the video element |
| Backend | FastAPI — coordinates Whisper + DeepFace + Translation calls |
| Hosting | Vercel (frontend) + Modal or Render (backend) |

### Key Notes
- Process audio in short chunks (e.g., 3-second windows) rather than true streaming — simpler and still feels real-time
- Emotion label refreshes every few seconds independently of transcription
- Visually the most striking demo — subtitle + emotion tag overlay on live video is immediately impressive

## Demo Project 6 — AI Dungeon Master

### Concept
User uploads any one-page TTRPG ruleset PDF. Claude becomes a fully playable dungeon master for that specific game — tracks state, rolls dice, narrates outcomes, responds to player choices.

### Tech Stack
| Layer | Tool |
|-------|------|
| Frontend | Next.js — chat-style game UI, dice roll animations (CSS or Framer Motion) |
| Ruleset Ingestion | LangChain + FAISS or Chroma — RAG pipeline over the uploaded ruleset PDF |
| LLM | Claude API — dungeon master persona, in-character narration, rules enforcement |
| State Management | Supabase table or Redis — tracks inventory, health, story beat, session history |
| Dice Logic | JavaScript on frontend — visual dice roll, result passed back to Claude as context |
| Backend | FastAPI — RAG retrieval + Claude orchestration |
| Hosting | Vercel (frontend) + Render (backend + vector store) |

### Key Notes
- RAG over the ruleset keeps Claude grounded in the specific game's rules rather than hallucinating
- State passed in every Claude call as a JSON blob — no memory needed between turns
- Niche but highly viral in gaming/indie dev communities — strong Reddit/Twitter demo potential
